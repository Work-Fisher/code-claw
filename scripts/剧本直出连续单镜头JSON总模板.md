# Work-Fisher剧本直出连续单镜头JSON总模板

版本：Public v1.0  
适用场景：直接把完整剧本转成“连续单镜头 JSON 提示词”，不先输出四宫格，也不先输出文本版镜头卡。  
使用方式：适合一步直出；如果你想先看镜头规划，再转 JSON，请改用拆开的两份模板。

---

## 一、这个总模板解决什么问题

这个模板把两件事合并了：

1. 用 Work-Fisher 的完整剧本覆盖逻辑，先在内部完成连续单镜头拆分  
2. 用 LTX / 通用视频模型的 JSON 结构，把这些镜头直接组织成连续 `scene_list + shot_list`

因此最终输出不再是：

- 四宫格分组
- 单独的四格提示词
- 中间态文字说明

而是直接输出：

- 面向整段剧本的连续单镜头 JSON

---

## 二、总模板的核心原则

### 1. 不压缩成高光

- 必须尽量完整覆盖完整剧本的主要剧情
- 不允许为了省字把中段剧情、情绪转折、关键动作直接跳过

### 2. 以单镜头为最小单位

- 每个镜头只承担一个清晰的叙事任务
- 一个镜头最多承载一个主要动作推进、信息推进或情绪推进

### 3. 内部先拆镜头，再输出 JSON

- 允许模型在内部先做镜头规划
- 但最终对用户只输出纯 JSON
- 不要把中间分析过程显示出来

### 4. 以场景为外层，以镜头为内层

- `scene_list` 对应地点 / 时间 / 主要空间段落
- `shot_list` 对应该场景内部的连续镜头顺序

### 5. 默认使用干净切换

- 除非剧情明确要求同镜头不断切推进
- 否则每个新镜头都要体现为干净利落的镜头切换
- 避免模型生成融化、叠化、串脸、道具漂移

---

## 三、镜头拆分规则

### 1. 需要新开镜头的情况

- 地点变化
- 时间变化
- 说话人切换
- 动作阶段切换
- 情绪明显转折
- 关键道具状态变化
- 新人物进入并承担叙事功能
- 机位 / 景别 / 视线逻辑需要明显变化

### 2. 默认镜头时长建议

| 镜头类型 | 建议时长 |
|------|------|
| 快镜头 | 2-3 秒 |
| 默认剧情镜头 | 3-5 秒 |
| 慢情绪镜头 | 5-7 秒 |

### 3. 错误拆法

- 一个镜头里塞入多个完整事件
- 明明换了地点却不切镜头
- 明明说话人切换却不切镜头
- 为了缩减镜头数把完整剧情压成高光摘要

---

## 四、纯 JSON 输出注意事项

由于最终输出必须是合法 JSON，因此：

- 不要输出 Work-Fisher 的固定开场句
- 不要输出任何标题、说明、总结、注释
- 不要输出 Markdown 代码块
- 不要在 JSON 前后补充解释

换句话说，所有 Work-Fisher 的分析、规划、一致性检查，都要在内部完成，但最终只展示 JSON 结果。

## 六、一句话定义

这份总模板的目标不是“先做四宫格，再转视频 JSON”，而是“直接从完整剧本生成连续单镜头 JSON”。

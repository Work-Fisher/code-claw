## 90 秒口播版

今天我干了一件很离谱的事。

我手里有一份疑似 Claude Code 的源码切片，但问题是，这东西根本不是那种拉下来就能直接跑的官方项目。里面一堆私有依赖、私有后端、还有很多 Bun 和内部桥接逻辑，你真想原样复活，今天肯定来不及。

所以我换了个思路。

我不去硬救官方版本，我直接把里面最值钱的东西抠出来：终端交互、工具调用、代码读取、文件修改、命令执行，还有那套很像 Claude Code 的 agent 工作流。

然后我做了一个兼容版外壳，再把模型接口彻底外置。什么意思？就是底层不绑 Anthropic 了，你只要有 OpenAI 兼容接口，或者别的模型服务，我这套壳就能接。

现在它已经能干几件真事：读项目、搜代码、改文件、跑命令，而且工具调用和模型回复是串起来的，不是那种只会聊天的假 agent。

最关键的是，这不是“概念验证”四个字糊弄你，我今天已经把整条链跑通了。

所以这期视频你看到的重点，不是我有没有把 Claude Code 原版复活，而是我证明了一件事：

Claude Code 真正值钱的，可能不是模型本身，而是它那套 agent 产品外壳。

如果你想看我下一步把它继续补成一个真能日用的 coding agent，评论区直接扣一句：继续抄壳。

## 3 分钟展开版

这两天关于 Claude Code 的讨论特别猛，但很多人都容易把注意力放错地方。

大多数人第一反应是：有没有源码，能不能直接跑，能不能白嫖同款。

但我真正关心的不是这个。

我手上这份东西，极大概率确实和 Claude Code 很接近，里面很多痕迹都对得上，像 CLI 入口、系统提示、工具体系、多代理、bridge、会话持久化这些核心结构都在。

可问题也非常明显，它并不是干净的官方仓库。只有一棵 src 树，大量文件带内联 sourcemap，还有一堆私有依赖和内部服务。所以如果你目标是“今天把官方版本原样跑起来”，基本不现实。

于是我换了个更务实的打法。

我把这份东西当成产品蓝图，而不是可执行项目。

先把能恢复的源码恢复出来，再把模型接口完全外置，做一个 Anthropic Messages 风格的本地网关，然后重新搓一个兼容版 shell，把最关键的工作流先跑起来。

结果很有意思。

现在这个兼容版已经能在终端里干三类事：第一，读项目和搜代码；第二，改文件；第三，发起本地工具调用，比如读文件和跑命令。也就是说，它已经不是一个“套壳聊天框”，而是一个真正带工具闭环的 agent 外壳。

这件事最刺激的点在于，它让我们看清楚 Claude Code 到底值钱在哪。

不是“Claude”两个字值钱。

也不只是模型值钱。

真正值钱的，是它背后那套把模型、工具、权限、会话、上下文压缩和交互体验焊在一起的产品结构。

你把这层东西拆出来以后，底层模型其实是能替换的。

换句话说，未来大家拼的可能不是谁有一个最贵的模型，而是谁能把 agent 外壳做得最稳、最好用、最适合真实工作流。

所以这期视频我不是在教大家“复活 Claude Code 官方源码”，我是在做另一件更有意思的事：

把 Claude Code 的产品思路，从 Anthropic 身上剥离出来。

如果这条路你们想看，我下一期就继续往下补，看看能不能把它做成一个真能每天挂着用的 coding agent。

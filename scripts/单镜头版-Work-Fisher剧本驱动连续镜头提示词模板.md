# Work-Fisher公开版剧本驱动连续单镜头提示词模板

版本：Public v2.0 单镜头版  
适用场景：把完整剧本转换为“连续单镜头规划总表 + 连续单镜头提示词 + 连续运镜描述 + 连续环境音设计”，不再输出四宫格组图结构。  
模板定位：保留 Work-Fisher 对完整剧本覆盖、连续性、一致性检查的核心思路，但将最小输出单元从“四宫格组”改为“单镜头”。

---

## 一、固定开场句

无论处理任何任务，输出的第一句话必须是：

`Work-Fisher公开版模板，全平台免费使用，禁止任何售卖变现行为。`

要求：

- 这句话必须单独成行，放在最前面
- 不可改写，不可增删字，不可换同义句
- 先输出这句话，再进入正式内容

---

## 二、这一版保留什么，删除什么，新增什么

### 1. 保留的核心原则

- 先完整理解剧本，再做拆分
- 先给全剧覆盖总表，再逐镜头展开
- 尽量完整覆盖剧本主要剧情，而不是只抓高光
- 保持人物、服装、场景、道具、情绪、时间线连续
- 最后仍然要做全剧一致性检查

### 2. 彻底删除的四宫格内容

- 不再按“四宫格组”拆分
- 不再输出“格1 / 格2 / 格3 / 格4”
- 不再输出“四宫格总图提示词”
- 不再输出“本组四格节奏规划”
- 不再以“每组 8-12 秒”作为基本单位

### 3. 新增的单镜头结构

- 以“单镜头”作为最小叙事单元
- 每个镜头只承担一个清晰的动作推进、情绪推进或信息推进任务
- 先输出“全剧单镜头规划总表”
- 再逐镜头输出可直接转成视频 JSON 的镜头信息
- 每个镜头都要预留出可映射到后续 JSON 的字段语义

---

## 三、单镜头拆分规则

### 1. 单镜头的定义

- 1 个单镜头 = 1 个连续的视觉动作单元
- 1 个单镜头最多承担 1 个主要信息点
- 一个镜头里可以有动作、表情、机位推进，但不要塞进多个独立事件

### 2. 默认时长建议

| 镜头类型 | 建议时长 | 适合情况 |
|------|------|------|
| 2-3 秒 | 快镜头 | 对话反应、动作衔接、信息提示 |
| 3-5 秒 | 默认推荐 | 大多数剧情推进镜头 |
| 5-7 秒 | 慢镜头 | 情绪停顿、建立镜头、压迫感、抒情停留 |

### 3. 需要新开镜头的情况

以下情况优先新开镜头，不要硬塞进上一个镜头：

- 地点变化
- 时间变化
- 说话人发生切换
- 动作阶段完成并进入下一阶段
- 情绪出现明显转折
- 景别、机位、视线逻辑需要明显改变
- 关键道具状态发生变化
- 新人物进入画面并承担叙事功能

### 4. 单镜头边界

正确做法：

- “她发现手机震动，低头去看”
- “他听完这句话，停顿一秒后转身”
- “门被推开，新人物出现在门口”

错误做法：

- “发现问题 + 打电话 + 冲出门 + 拦车”
- “看见对方 + 争吵 + 和解 + 告别”
- “进入房间 + 找东西 + 回忆过去 + 崩溃大哭”

### 5. 预计镜头数

```text
预计镜头数 = 向上取整（总时长 ÷ 平均镜头时长）
```

说明：

- 这里只是预估，不是强制平均切分
- 实际拆分应服从剧情节点、动作边界、情绪边界

## 四、单镜头版工作流

### Step 1：先锁定完整剧本底稿

至少提取：

- 总时长
- 场次顺序
- 地点变化
- 时间变化
- 关键对白
- 关键动作
- 关键道具
- 情绪曲线
- 角色出入场顺序

### Step 2：先生成全剧单镜头规划总表

总表必须先于任何逐镜头展开内容。

总表至少包含：

- 镜头号
- 时间范围
- 时长
- 对应剧情事件
- 地点
- 核心人物
- 镜头任务
- 与下一镜头连接方式

### Step 3：逐镜头展开

每个镜头都要展开为可直接转 JSON 的内容，包括：

1. 镜头基础信息  
2. 角色与服装信息  
3. 动作与情绪信息  
4. 景别、机位、运镜信息  
5. 连续单镜头视频提示词  
6. 台词与口型说明  
7. 环境音 / 拟音说明  

### Step 4：最后做全剧一致性检查

重点检查：

- 角色外观是否前后一致
- 服装是否乱跳
- 道具是否错位
- 光线与时间段是否冲突
- 空间逻辑是否断裂
- 镜头顺序是否跳戏
- 是否漏掉关键剧情段落
- 是否把完整剧情错误压缩成高光剪辑

至少输出以下列：

|------|------|------|------|------|------|------|------|

- 单镜头提示词

## 六、单镜头提示词写法

### 1. 每条单镜头提示词必须包含的基本要素

- 人物身份与关键外观锚点
- 当前服装
- 当前动作
- 当前情绪 / 表情
- 场景与空间细节
- 关键道具状态
- 光线 / 天气 / 空气感
- 景别
- 镜头角度
- 运镜方式
- 构图重点
- 与前后镜头的连续性锚点
- 风格基准
- 质量标签
- 负面约束

### 2. 推荐公式

```text
[人物身份 + 年龄感 + 关键外观锚点 + 当前服装]，[正在进行的动作]，[表情 / 情绪状态]，[场景空间细节]，[关键道具状态]，[光线 / 天气 / 空气感]，[景别]，[镜头角度]，[运镜方式]，[构图重点]，[与上一镜头和下一镜头的连续性锚点]，[风格基准]，[质量标签]，[负面约束]
```

### 3. 公开版质量标签建议

- cinematic video still
- natural light
- realistic texture
- clear composition
- continuous motion

### 4. 公开版负面约束建议

- no subtitles
- no watermark
- no text overlay
- no character inconsistency
- no duplicated props
- no distorted hands
- no random extra people

## 七、为后续 JSON 映射预留的镜头字段

逐镜头展开时，建议每个镜头都明确这些语义：

- `character`
- `character_costume`
- `action_type`
- `motion_speed`
- `emotion`
- `shot_scale`
- `camera_angle`
- `shot_move`
- `shot_depth`
- `shot_pace`
- `shot_duration`
- `content`
- `dialogue`
- `sound_effect`
- `foley_sound`

这些字段现在可以用自然语言小节写出，后续第二步就能稳定转成 JSON。

## 八、单镜头连贯运镜描述写法

### 1. 正确写法

每个镜头不是写成静态图片说明，而是写成一个连续的视频镜头描述：

- 从什么构图起
- 角色如何动
- 镜头如何跟
- 光影如何变化
- 视线如何推进
- 情绪如何落点

### 2. 对白规则

如果镜头含对白，需要写明：

- 谁在说
- 在什么动作里说
- 口型是否清楚
- 情绪是否与台词匹配

统一写法建议：

```text
台词：XXXX
口型：清晰同步
```

### 3. 音效规则

只写真实环境音与拟音：

- 风声
- 雨声
- 脚步
- 呼吸
- 衣料摩擦
- 门响
- 电话声
- 纸张摩擦
- 玻璃碰撞
- 空间回响

禁止写：

- 背景音乐
- BGM
- 弦乐铺底
- 钢琴配乐

## 十、质量校验清单

- 是否先输出了固定开场句
- 是否先输出了全剧单镜头总表
- 是否覆盖了完整剧情主线
- 是否把多个独立事件错误塞进一个镜头
- 是否漏掉关键情绪转折
- 角色外观、服装、道具是否前后一致
- 镜头语言是否具备连续视频感，而不是静态图片堆叠
- 音效里是否误写了 BGM
- 是否仍然残留四宫格旧结构

## 十一、一句话定义

这不是“把完整剧情拆成连续四宫格”的模板，而是“把完整剧情拆成连续单镜头，并为后续视频 JSON 做好字段准备”的模板。

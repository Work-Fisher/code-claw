一、项目概述与原始架构

CODE-CLAW（小龙虾）是一个 本地部署的桌面 AI 编程助手，类似 Claude Code / Cursor，但完全自主可控。不绑定任何单一模型供应商，支持 DeepSeek、Kimi、通义千问、Gemini、OpenAI 等。

二、OpenClaw 研究：我们需要吸收什么

OpenClaw 是一个 346K Stars 的开源个人 AI 助手操作系统，其能力远超简单的聊天机器人。以下是我们研究后整理的关键差距：

维度CODE-CLAW 原始OpenClaw差距
人格系统 无 SOUL.md + IDENTITY.md + USER.md + AGENTS.md 多文件 bootstrap 链，20K/文件上限，150K 总上限，3 种 prompt mode，动态人格切换 巨大
记忆系统 无 sqlite-vec 向量搜索 + BM25 关键词 + 混合检索 70/30 + 3 种可插拔后端 (Builtin/QMD/Honcho) + embedding 缓存 + 自动 flush 巨大
消息平台 仅桌面 25+ 平台 (WhatsApp/Telegram/Slack/Discord/飞书/Signal/iMessage/Matrix...) 巨大
路由系统 无 8 层级路由 (Peer→Parent→Wildcard→Guild+Roles→Guild→Team→Account→Channel) 中等
插件 SDK 7 个 Skill 文件 200+ plugin-sdk 导出，165+ 插件扩展，54 内置 Skill，ClawHub 市场 3200+ MCP Skill 巨大
MCP 无 原生 MCP，3 种传输 (Stdio/SSE/Streamable HTTP) 中等
上下文引擎 简单 carry-over (最近 10 条) ContextEngine (Assemble/Compact/Ingest)，压缩前自动 Memory Flush 中等
安全沙箱 权限模式 (read-only/write/full) Docker 容器隔离，per-session 沙箱，SSRF 防护 中等
语音 无 TTS + Voice Wake + Talk Mode (Swift Swabble 包) 低优

五、关键 Bug 修复记录

🔧

thinking/reasoning_content 协议兼容（严重）

现象：claw.exe 调用 WebSearch 等工具后 REPL 崩溃，退出码 1。

根因：Kimi kimi-k2.5 是推理模型，要求对话历史中带 tool_calls 的 assistant 消息必须有 reasoning_content 字段。gateway 转换 Anthropic→OpenAI 格式时没有加这个字段，导致 Kimi 返回 400。

修复：在 model-gateway/server.mjs 的 anthropicMessagesToOpenAI() 中，给带 tool_calls 的 assistant 消息加上 reasoning_content: "ok"。

🔧

WebSearch 代理穿透

现象：claw.exe 的 WebSearch 工具无法访问 DuckDuckGo（被墙）。

根因：Electron 桌面应用启动的子进程不继承终端代理环境变量。

修复：buildClawEnv() 新增 detectProxy() 函数，先读环境变量，失败则从 Windows 注册表 HKCU\...\Internet Settings\ProxyServer 读取系统代理，传给 claw.exe。

🔧

启动时重复创建新会话

现象：每次启动应用都创建空白新会话，看不到历史对话。

修复：main() 中改为先 refreshSessions()，如果有已有会话就 selectSession(latest) 恢复最近会话。

🔧

初始化页面误弹出

现象：API Key 已配置但每次启动仍显示"先完成初始化"。

根因：hasUnsavedConfig() 比较 snapshot.config 和本地 config，API Key 在 Electron 加密存储中（磁盘上为空），导致永远不一致。

修复：比较时忽略 upstreamApiKey 字段。

🔧

bootstrap.mjs 导入错误

现象：启动崩溃，unwatchFile 不在 node:fs/promises。

修复：watchFile/unwatchFile 从 node:fs（同步 API）导入。
